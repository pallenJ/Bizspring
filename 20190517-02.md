## SQL Injection
SQL 인젝션(SQL 삽입, SQL 주입으로도 불린다)은 코드 인젝션의 한 기법으로 클라이언트의 입력값을 조작하여 서버의 데이터베이스를 공격할 수 있는 공격방식을 말한다. 주로 사용자가 입력한 데이터를 제대로 필터링, 이스케이핑하지 못했을 경우에 발생한다.

#### 공격방법
아이디와 비밀번호를 받아 사용자 정보를 받는 SQL문이 있다고 가정한다.
```sql
SELECT user FROM user_table WHERE id='입력한 아이디' AND password='입력한 비밀번호';
```
아이디 : user01, 비밀번호 : pw01로 로그인을 가정할 시,
```SQL
SELECT user FROM user_table WHERE id='user01' AND password='pw01';
```
하지만 악의적인 사용자가 다음과 같이 입력한다.
```
아이디 : admin
비밀번호 : ' OR '1' =' 1
```
```SQL
SELECT user FROM user_table WHERE id='admin' AND password=' ' OR '1' = '1';
```
따옴표를 올바르게 닫으며 password=' '를 만들어 버림과 동시에 SQL 구문 뒤에 OR '1' = '1'을 붙였다. 즉 아이디와 비번을 제대로 매치하지 못했어도 ' OR '1' = '1 구문 때문에 로그인에 성공하게 되는 것이다.

#### 방어방법
1. 대부분의 SQL Injection의 경우 값을 입력받을 때 특수문자 여부를 검사하여 방어한다.  
아이디와 패스워드를 입력받는 입력에 특수문자가 포함되어 있는지 검증 로직을 추가하여 특수문자가 입력된 경우 해당 요청을 막아낼 수 있다.

2. Statement대신 preparestatement를 사용하자.  
mysql의 Preparedstatement 구문을 사용하면 특수문자를 자동으로 escaping 해준다. 클라이언트 측 자바스크립트로 폼 입력값을 우선 검증하자. 하지만 자바스크립트는 공격자가 꺼버리면 그만이기 때문에 서버 측에서 입력값을 한 번 더 필터한다.


## CSRF(Cross-Site Request Forgery)
웹 애플리케이션 취약점 중 하나로 사용자가 자신의 의지와 무관하게 공격자가 의도한 행동을 해서 특정 웹페이지를 보안에 취약하게 한다거나 수정, 삭제 등의 작업을 하게 만드는 공격 방법을 의미한다.  
XSS공격과의 차이점이 있다면 XSS공격은 공격대상이 Client고 CSRF는 서버이다.

## 암호화 알고리즘
* MD5(Message-Digest algorithm 5) - 128비트 암호화 해쉬함수, 해킹에 취약(SHA-1 권장), 무결성 검사에 많이 사용

* SHA(Secure Hash Algorithm) - 160비트 암호화 해쉬함수, MD4의 발전 형태, 암호학적 해쉬함수들의 모음

* RSA - 공개키 암호시스템의 하나, 전자서명이 가능한 최초의 알고리즘 -> 전자상거래등에 활용  
원본 -- *공개키* -- 암호화 -- 유통 --> 암호화 데이터 -- *개인키* --> 원본

* AES - 암호화, 복호화 과정에서 동일키를 사용하는 대칭키 알고리즘, 128비트
